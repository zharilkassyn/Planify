# 🤖 Промпты для Codex

Codex меняет **настоящий код** в твоём проекте. Чтобы он делал это правильно — дай ему контекст
один раз, а потом проси фичи по шаблонам. Всё в `[квадратных скобках]` — заменяй на своё.

---

> 💡 В проекте есть файл **AGENTS.md** — Codex читает его сам и уже знает стек, правила и команды.
> Контекст-промпт ниже — короткое напоминание на случай, если ассистент его не подхватил.

## 0️⃣ Контекст-промпт (вставь ПЕРВЫМ, один раз в начале проекта)

```
Это мой проект на стеке: Vite + React + TypeScript + Supabase + Vercel.
Следуй правилам из AGENTS.md. Коротко:
- Supabase уже подключён в src/lib/supabase.ts (ключи в .env.local). Используй ЕГО.
- Базу меняй ТОЛЬКО миграциями: npm run db:new -- имя → впиши SQL → я запущу npm run db:push.
  Никакого ручного SQL Editor. На каждую таблицу — RLS «только свои строки».
- Код на TypeScript, без any. Маленькие компоненты. Перед концом — npm run build.
- Не трогай ключи и .env, не коммить .env.local.
- Объясняй кратко, что поменял и в каких файлах.
Подтверди, что понял, и жди мою первую задачу.
```

---

## 🔁 Главное правило цикла

После КАЖДОГО промпта Codex: **посмотри что изменилось → запусти `npm run dev` → работает? →
`git push`**. Сломалось — попроси починить, не коммить сломанное.

## 🗓️ Промпты по дням (шаблоны)

### День 3 — первая своя фича
```
Переделай стартовый экран под мою идею: [опиши идею в 1-2 предложениях].
Замени пример «записи» на [что у тебя: вопросы квиза / привычки / места / карточки].
Оставь вход и подключение к Supabase как есть. Создай новую миграцию
(npm run db:new -- имя) под новые поля и впиши туда SQL — я запущу npm run db:push.
```

### День 4 — база + вход + профиль
```
Добавь страницу профиля: показывай email, аватар (буква имени) и статистику —
[например: сколько [записей] создано, дата регистрации].
Сделай так, чтобы [мои данные: счёт игры / прогресс / список] сохранялись в Supabase и
подгружались при входе. Для новых таблиц создавай миграции (npm run db:new), включай
RLS «только свои строки». Я применю npm run db:push.
```

### День 5 — настоящий AI внутри продукта (Gemini)
```
Добавь AI-функцию: [что должен делать AI — например: генерировать 5 вопросов по теме /
давать совет по моим данным / отвечать в роли персонажа].
Используй уже готовую Edge Function 'ai' (Gemini) — вызывай её через
supabase.functions.invoke('ai', { body: { prompt, system } }) и показывай data.text.
Не создавай новую функцию и не вызывай модель с фронта напрямую.
System prompt (поле system): роль — [роль], тон — [тон], запреты — [чего нельзя], формат — [формат].
```
> Перед этим (один раз): бесплатный ключ на aistudio.google.com/apikey →
> `npm run ai:secret -- GEMINI_API_KEY=...` → `npm run ai:deploy`.

### День 6 — красота + удержание
```
Сделай дизайн дорогим и спокойным: один акцентный цвет [#цвет], тёмный текст, много воздуха,
шрифт Inter, скруглённые кнопки, без неона. Применить ко всем экранам.
Добавь крючок возвращения: [streak по дням / XP и уровни / таблица лидеров / ежедневный челлендж].
Если у меня игра — игровой процесс не трогай, меняй только меню и экраны вокруг.
```

### День 7 — реальные пользователи + онбординг
```
Добавь онбординг: welcome-экран в одно предложение «что это», подсказку на главной кнопке и
дружелюбные пустые экраны (что делать, когда данных ещё нет).
Сделай так, чтобы по моей ссылке любой человек мог зайти и сразу понять за 30 секунд, что делать.
```

### День 8 — правки (без новой темы)
```
Вот баг: [что не так и что должно быть]. Найди причину и почини минимально, не ломая остальное.
После — проверь, что npm run build проходит.
```
